# Телефонная книга предприятия (AD Phone Book)

Десктопное WPF-приложение для просмотра контактов сотрудников из Active Directory.

---

## Требования

| Компонент        | Минимальная версия          |
|------------------|-----------------------------|
| ОС               | Windows 10 / Windows Server 2016 |
| .NET SDK         | 6.0 (или 7/8 — совместимы) |
| Visual Studio    | 2022 (Community достаточно) |
| Членство в домене| Машина должна быть в домене AD (или иметь доступ к DC) |

---

## Быстрый старт

### 1. Клонирование / распаковка
```
Распакуйте архив или скопируйте папку ADPhoneBook на рабочую машину
```

### 2. Сборка через Visual Studio
1. Откройте `ADPhoneBook.sln` в Visual Studio 2022
2. `Сборка → Собрать решение` (Ctrl+Shift+B)
3. `Debug → Запустить` (F5)

### 3. Сборка через командную строку
```cmd
cd ADPhoneBook
dotnet restore
dotnet build -c Release
dotnet run --project ADPhoneBook
```

### 4. Публикация (standalone EXE без установки .NET)
```cmd
dotnet publish ADPhoneBook -c Release -r win-x64 --self-contained true ^
  -p:PublishSingleFile=true -o .\publish
```
Готовый файл: `publish\ADPhoneBook.exe` — запускается на любой Windows-машине без установки .NET.

## Первый запуск

1. Запустите `ADPhoneBook.exe`
2. Если машина в домене — нажмите **«Загрузить»** сразу (используется текущий пользователь)
3. Если нужны кастомные настройки — нажмите **«⚙ Настройки»** и укажите LDAP-путь и учётные данные

## Настройки подключения

| Поле               | Описание                                                  | Пример                                  |
|--------------------|-----------------------------------------------------------|-----------------------------------------|
| LDAP путь          | Корень для поиска. Пусто = текущий домен                  | `LDAP://DC=company,DC=local`            |
| Использовать текущего пользователя | Авторизация через Kerberos текущей сессии | ✅ рекомендуется                        |
| Логин              | Если не используем текущего пользователя                  | `COMPANY\john.doe` или `john@company.local` |
| Пароль             | Пароль учётной записи                                     | (не сохраняется на диск)                |

> **Важно:** Пароль хранится только в памяти приложения и не записывается в файл настроек.

### Примеры LDAP-путей
```
LDAP://DC=company,DC=local               — корень домена
LDAP://OU=Employees,DC=company,DC=local  — только определённый OU
LDAP://192.168.1.10/DC=company,DC=local  — подключение по IP контроллера домена
```

## Функциональность

### Загрузка данных
- Загружает всех **активных** пользователей (отключённые аккаунты исключаются)
- Поддерживает пагинацию LDAP (1000 записей за раз) — работает с большими доменами
- Асинхронная загрузка — UI не блокируется
- Кнопка «Отмена» для прерывания долгой загрузки

### Поиск
- Мгновенный поиск по: ФИО, отделу, должности, email, рабочему и мобильному телефону
- Поле очищается крестиком

### Фильтрация по отделам
- Выпадающий список заполняется автоматически из загруженных данных
- «Все отделы» — показать всех сотрудников

### Экспорт
- **Excel (.xlsx)** — форматированная таблица с заголовком, чередующимися строками, автофильтром и закреплением первой строки
- **CSV (UTF-8 BOM)** — открывается напрямую в Excel с корректной кодировкой

> Экспортируются только **отфильтрованные** записи (то, что видно в таблице).

## Частые вопросы

**Q: Приложение запускается, но список пустой**  
A: Убедитесь, что машина в домене. Нажмите «⚙ Настройки» и проверьте LDAP-путь. Проверьте, что используемая учётная запись имеет права на чтение AD.

**Q: Ошибка "The server is not operational"**  
A: Контроллер домена недоступен. Укажите IP/имя DC явно в поле LDAP: `LDAP://dc01.company.local/DC=company,DC=local`

**Q: "Access is denied"**  
A: Текущий пользователь не имеет прав на чтение AD. Снимите галку «Использовать текущего пользователя» и введите учётные данные сервисной учётки с правами чтения AD.

**Q: Загружает слишком медленно (10 000+ пользователей)**  
A: Ограничьте область поиска, указав конкретный OU в LDAP-пути вместо корня домена.
